# Focus Translator

右Ctrlキーを押すとマウスポインタ直下のテキストを認識して、翻訳をカーソル近傍にオーバーレイ表示する常駐ツールです。

## 開発経緯

PCで海外のソフトウェアを使ったり、英語のドキュメントやWeb記事を読んだりしている際、わからない単語や文章が出てくるたびに「テキストを選択してコピーし、ブラウザや別の翻訳ツールに貼り付ける」という作業を繰り返すことに煩わしさを感じていました。
また、従来の画面翻訳ツールは画面全体を翻訳してしまうものが多く、元のレイアウトが崩れたり、本当に知りたい部分以外も翻訳されてしまって逆に読みづらくなることがありました。

そこで、「ユーザーの現在の視線を一切動かさず、知りたいと思ったその瞬間に、マウスカーソルが指している場所だけをピンポイントで翻訳・解説してくれるツールが欲しい」と考えたのが FocusTranslator の開発のきっかけです。

右Ctrlキーを押している間だけサッと翻訳が表示され、離せばすぐに元の作業に戻れる。画像として表示されているテキストであってもOCRで瞬時に読み取れる。さらに「ただ翻訳するだけでなく、生成AI（LLM）の力でその文脈や専門用語の解説までしてくれる」という、読書や作業のフォーカス（Focus）を一切削ぐことのない、シームレスな翻訳体験を目指して開発しました。

## ビルド

```
cargo build --release
```

生成物: `target\release\focus-translator.exe`。ONNX Runtimeは静的リンクのため通常動作(CPU推論)に外部DLLは不要。`DirectML.dll` は同梱されるが遅延ロードのDirectML実行プロバイダ用で、未使用のため無くても起動・動作する。

要件: Rust stable (x86_64-pc-windows-msvc), Windows 11

## 設定・モデルの保存先

既定では `%APPDATA%\FocusTranslator\` (config.json, models\, ログ) に保存される。

## 使い方

| 操作                | 動作                                                              |
| ------------------- | ----------------------------------------------------------------- |
| **右Ctrl ホールド** | カーソル下の1行を認識・翻訳して表示。離すと消える                 |
| **Ctrl+Alt+T**      | 範囲指定モード。ドラッグ選択した矩形をOCR・翻訳(最初からピン留め) |
| **チップ押下**      | OCR/翻訳エンジンを切替えて再処理。押下時点でピン留め              |
| **コピー**          | 訳文(なければ原文)をクリップボードへ                              |
| **Esc / 閉じる**    | ピン留め表示を閉じる                                              |
| **タスクトレイ**    | 設定画面 / 範囲指定翻訳 / 終了                                    |

## 画面一覧と詳細解説

FocusTranslatorは、タスクトレイ常駐型アプリとして以下の主要なUI画面を備えています。

### 1. オーバーレイ画面（翻訳結果表示）

キャプチャ直後にカーソル付近にポップアップするメイン画面です。

- **基本構成**: 上段に小さく「認識した原文」、下段に大きく「翻訳結果」を表示します。
- **チップ（ボタン群）**: 結果の下に並ぶボタンで、エンジン切替（別のOCRや翻訳エンジンで再処理）、解説（LLMによる詳細な意味や用途の解説）、音声再生（テキストの読み上げ）、キャプチャ編集、コピーなどの操作が可能です。
- **テキストの直接修正**: 原文や訳文、解説文の横にある「鉛筆アイコン」をクリックすると、別ウィンドウが開いて誤認識されたテキストを手動で修正できます。原文を修正した場合は、自動的に再翻訳が行われます。
- **ピン留め（固定）機能**: ホールドキーを設定秒数（既定3秒）長押しするか、ウィンドウ内をドラッグ操作することで、キーを離しても画面が消えずに残り続ける状態（ピン留め）になります。

### 2. キャプチャ編集画面

オーバーレイ画面の【キャプチャ画像】ボタンから展開する、インラインの画像編集機能です。

- **機能**: 読み取った画面キャプチャをトリミング（切り抜き）し、不要な背景ノイズを減らしてOCR精度を向上させます。
- **操作**: マウスドラッグによる「矩形トリミング」と「投げ輪（ポリゴン）トリミング」に対応しています。広範囲をキャプチャした場合は自動で画面内に縮小表示されます。【選択範囲を残す】（切り抜き）や【選択範囲を消す】（周辺色で塗りつぶし）などを適用すると、その画像を使って再度OCRが実行されます。元に戻す（Undo）操作にも対応しています。

### 3. 設定画面

タスクトレイメニューから開く、横長3列レイアウトの総合設定画面です。

- **左列（操作・OCR設定）**: ホットキーやピン留め長押し時間の変更、既定OCRエンジンの選択など。
- **中列（翻訳設定・その他）**: 既定翻訳エンジン、翻訳方向、APIキー（DeepL / Google / Gemini等）の設定、ログ記録のON/OFFなど。
- **右列（LLMプロファイル設定）**: 生成AI（LLM）を使用する際のプロファイルを複数管理できます。翻訳、OCR統合、解説用のプロンプトを個別に設定可能です。

### 4. プロンプト編集ウィンドウ

設定画面の各プロンプトボタン、またはオーバーレイの【解説】チップから展開する画面です。

- **変数一覧の活用**: 左側に `{{source_lang}}` や `{{original_text}}` などの「利用可能なプレースホルダ（変数）」がリスト表示されており、ダブルクリックで簡単にテンプレートへ挿入できます。
- **プレビューと送信**: オーバーレイの解説機能から呼び出した場合は、テンプレートに変数が展開された「実際の送信内容」がプレビュー表示され、内容を手動で微調整してからLLMへ送信することができます。

### 5. ログビューアー画面

タスクトレイメニューまたは設定画面から開く、履歴の閲覧・再実行画面です。

- **レイアウト構成**: 画面を「田の字」の2×2（上段: 入力内容 / 読み取り結果、下段: 翻訳結果 / 解説結果）に分割し、直感的に処理の工程を追えるツリー構造になっています。スプリッター（境界）はドラッグで自由にリサイズ可能です。
- **主要機能**:
  - 過去のテキストや画像履歴の閲覧（画像はクリックで原寸1:1表示）。
  - 過去の入力に対する「再OCR」「再翻訳」「再解説」の実行と比較。
  - 【入力追加】欄からの手動テキスト追加（マニュアル入力の翻訳）。
  - 全文検索やアプリ名（exe）による絞り込み、CSV出力機能。
- **データ保存**: ログ保存をONにした場合、データはローカルのSQLite (`%APPDATA%\FocusTranslator\logs\focustranslator.db`) に保存されます。APIキー等の機密情報はマスク処理されて記録されます。

## エンジン対応状況

| エンジン                      | 種別               | 状態                                                                                                                                                            |
| ----------------------------- | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Windows.Media.Ocr             | OCR (既定)         | ✅ 実装済み                                                                                                                                                     |
| YomiToku / NDL-OCR            | OCR (外部サーバー) | ✅ HTTPクライアント実装済み (`POST /ocr`, `GET /health`)                                                                                                        |
| Gemini                        | OCR+翻訳統合       | ✅ 実装済み (画像→原文+訳文一括)                                                                                                                                |
| PaddleOCR                     | OCR                | ⏳ モデル導入(設定画面からワンクリックDL+SHA256検証、[paddle_install.rs](src/paddle_install.rs))は実装済み。ONNX Runtime推論は次版                              |
| DeepL / Google Trans / Gemini | 翻訳               | ✅ 実装済み (REST)。設定画面に各APIキー取得ページを開くボタンあり                                                                                               |
| ローカルONNX翻訳              | 翻訳 (既定)        | ✅ 実装済み。opus-mt-ja-en / opus-mt-en-jap (ONNX量子化, [onnx_translate.rs](src/onnx_translate.rs)) による貪欲法(greedy)デコード。KVキャッシュ未使用の簡易実装 |

外部OCRサーバーの想定API: `POST {url}/ocr` (body: `image/png`) → `{"text": "..."}` または `{"results": [{"text": "..."}]}`、`GET {url}/health`。

## プライバシー (SPEC §9)

- 既定構成 (Windows OCR + ローカル翻訳) では外部送信なし。
- クラウド/外部サーバーエンジンの初回利用時に送信種別ごと(テキスト/画像/外部OCRサーバー)の同意ダイアログを表示。`127.0.0.1` へのサーバーURLはローカル送信として扱う。
- 同意なしにホールドモードの既定エンジンが外部送信することはない(未同意時はローカルエンジンへ自動代替)。
- APIキーは DPAPI で暗号化して保存。キー・送信テキスト・画像はログに出さない。設定画面の入力欄も ● でマスク表示。
- 計測ログ(設定で有効化)はステージ別所要時間のみ記録。

## モデルの導入(ワンクリックインストール)

設定画面の「PaddleOCR」「ローカルONNX翻訳」の各行で導入状況を確認でき、未導入時は「インストール」ボタンでワンクリック導入できる。いずれもダウンロード後にSHA256を検証し、一致した場合のみ配置する(不一致時は破棄しエラー表示)。

- **PaddleOCR**: 配布元は [RapidAI/RapidOCR](https://github.com/RapidAI/RapidOCR)(ModelScope)公開の PP-OCRv4 mobile 版 ONNX(検出/日本語認識/辞書の3ファイル、計約14MB)。`%APPDATA%\FocusTranslator\models\paddleocr\` に配置。OCRの推論(ONNX Runtime連携)は次版対応で、導入済みの状態でPaddleOCRチップを選択すると「推論は未実装です」と表示される。
- **ローカルONNX翻訳**: 配布元は [Xenova/opus-mt-ja-en](https://huggingface.co/Xenova/opus-mt-ja-en) / [Xenova/opus-mt-en-jap](https://huggingface.co/Xenova/opus-mt-en-jap)(Helsinki-NLP OPUS-MTのONNX量子化版)。日→英・英→日それぞれのencoder/decoder/tokenizer、計6ファイル・約200MB。`%APPDATA%\FocusTranslator\models\onnx_translate\` に配置。導入後は実際に推論が動作する。
